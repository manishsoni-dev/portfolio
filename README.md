# Manish Soni Portfolio

Professional developer portfolio for Manish Soni, built with Astro, Tailwind CSS, and TypeScript-friendly content collections.

## Stack

- Astro
- Tailwind CSS
- TypeScript-aware Astro components and content schemas
- Static deployment target for Vercel
- Markdown/MDX project content collection

## Local Commands

Run these from the project root:

```sh
npm install
npm run dev
npm run check
npm run build
npm run preview
npm run verify
```

The local dev server runs at `http://localhost:4321/` by default.
`npm run build` runs `astro check` before `astro build`, so Vercel builds include type checking.
`npm run verify` runs the build and then checks for accidental fabricated/preview content, missing profile links, and broken optional sections.

## Adding Real Projects

Add real project writeups as Markdown or MDX files in `src/content/projects/`.
The homepage only shows a Projects section when at least one non-draft `.md` or `.mdx` project exists.

Required frontmatter:

```yaml
---
title: "Project name"
summary: "Short factual summary."
stack:
  - Python
  - FastAPI
draft: false
---
```

Optional frontmatter:

```yaml
---
role: "Your role on the project"
repository: "https://github.com/..."
demo: "https://..."
featured: true
order: 1
published: 2026-01-01
---
```

Only add projects that can be described accurately. Avoid fake metrics, inflated ownership, or unsupported case studies.

## Adding a Resume

Place the resume PDF at:

```text
public/resume/manish-soni-resume.pdf
```

The homepage and contact section will only show a Resume link when that exact file exists.
Until then, the site intentionally hides resume CTAs so there are no broken links.

## Vercel Deployment

This is a static Astro site, so Vercel can auto-detect and deploy it without an adapter.
The included `vercel.json` keeps the build command and output directory explicit:

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist"
}
```

Deployment options:

```sh
npm install -g vercel
vercel
```

Or push the repository to GitHub and import it into Vercel.

### Production Site URL

After the first Vercel deployment, set this environment variable in Vercel:

```text
PUBLIC_SITE_URL=https://your-deployed-domain.vercel.app
```

Use the real production domain, including `https://`.
When `PUBLIC_SITE_URL` is set, the site emits canonical URLs, `og:url`, and absolute social preview image URLs.
When it is not set, local builds stay domain-neutral and skip canonical URLs.
The sitemap integration follows the same rule: sitemap files are generated only when `PUBLIC_SITE_URL` is set.
`robots.txt` is generated by Astro and adds a `Sitemap:` line only when `PUBLIC_SITE_URL` is set.
