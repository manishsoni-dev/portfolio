## Problem

Financial fraud detection requires processing high-volume transaction streams with low latency and high recall — missing a fraudulent transaction is costly, but flagging too many legitimate ones degrades user trust.

## Approach

Built a classification pipeline using scikit-learn that trains on labeled transaction data, engineers features from transaction metadata (amount patterns, frequency, geolocation anomalies), and serves predictions via a REST API endpoint.

## Key Details

- Feature engineering pipeline extracts temporal patterns, velocity checks, and statistical anomalies from raw transaction records
- Evaluated multiple classifiers (Random Forest, Gradient Boosting, Logistic Regression) with cross-validation
- Handled severe class imbalance using SMOTE and adjusted class weights
- REST API accepts transaction payloads and returns fraud probability scores with configurable thresholds
