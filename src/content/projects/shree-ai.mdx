## Problem

Developers and teams waste time evaluating dozens of overlapping AI tools when they need a specific capability. There was no fast, semantic way to describe a need in plain English and get matched to the right tool.

## Approach

Built a FastAPI backend that ingests tool metadata, generates embeddings for each tool's description and capabilities, and stores them in a FAISS vector index. When a user submits a natural-language query, the system embeds the query and returns the top-k most semantically similar tools with relevance scores.

## Key Details

- Designed the ingestion pipeline to normalize tool metadata from multiple sources into a consistent schema
- Used embedding-based vector generation for semantic matching
- FAISS index enables sub-millisecond similarity search across the full tool corpus
- FastAPI serves the search endpoint with automatic OpenAPI documentation
